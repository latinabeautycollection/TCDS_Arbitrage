# TCDS Green Tier 1 Enterprise Design System
## Domain 6 Warehouse PWA Shell v2.0

## Design Goal

The shell must look and feel like a TCDS enterprise product, not a generic inventory SaaS application.

The product identity is built around:

- Precision
- Security
- Executive quality
- Minimal warehouse friction
- High readability
- Barcode-first execution

## Primary Visual Language

Use:

- White surfaces
- Black command areas
- Gold emphasis
- Muted gray metadata
- Subtle green success states
- Minimal red for true exceptions only

Avoid:

- Blue-heavy SaaS styling
- Decorative gradients that reduce readability
- Excessive icons
- Dense cards
- Too many competing colors

## Color Rules

Gold is reserved for:

- Primary emphasis
- Scan frames
- Active navigation
- Important status highlights
- Brand identity
- Focus rings

Black is reserved for:

- Primary buttons
- Command-center headers
- High-authority panels
- Bottom system strip

White is reserved for:

- Main content surfaces
- Forms
- Cards
- Lists

## Typography Rules

Every screen follows a strict hierarchy:

1. Page title: large, bold, high contrast
2. Section title: medium, bold
3. Operational text: readable body size
4. Metadata: muted, small, not competing

## Motion Rules

Motion must be subtle and purposeful:

- Button press feedback
- Card press feedback
- Scanner pulse
- Future page transitions
- Future success check animation
- Future loading skeletons

No flashy animation is allowed.

## Screen Rules

The 12-screen architecture is frozen.

Additional functions must use:

- Bottom sheets
- Toasts
- Dialogs
- Expandable panels
- Inline states

No new production screen may be added without architectural approval.
